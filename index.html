Real-World Application

Built for
active research

Koniscope was created and deployed in the field — not as a demo, but as the primary analysis tool for an ongoing scientific investigation into manatee health.

🐾

Manatee Cachexia Detection

Koniscope identifies early signs of cachexia — progressive muscle wasting — in Florida manatees (Trichechus manatus latirostris) along the West Coast of Florida by measuring fiber cross-sectional area distributions in necropsy H&E slides.

📊

Morphometric Statistics

Each run produces a six-panel morphometry dashboard: fiber diameter, area, circularity, aspect ratio, and scatter plots — along with IQR, SD, and full export-ready CSV output.

🔬

Full SVS Whole-Slide Auto-Parsing

Koniscope automatically tiles and processes entire SVS whole-slide images across all available CPU cores in parallel. A single full-resolution SVS scan containing 16,000+ muscle fibers is detected, segmented, and precisely marked with no manual cropping, no tile selection, and no user intervention — just load the file and run.

⚡

Rebuilt Segmentation Engine

Version 2.0 corrected inverted thresholding logic, added per-fiber crack filling for internal myofibril artifacts, and introduced adaptive watershed parameters tuned to large-fiber tissue.

Segmentation Engine v2.0

7-step detection pipeline

Every fiber is detected, crack-filled, split where necessary, and validated by shape — without any manual parameter tuning per slide.

01

Luminance Thresholding

Bright connective tissue and background (high L channel in LAB space) is excluded. Fiber bodies remain as the low-L foreground mask.

LAB · L-channel
02

Morphological Opening

A 7px elliptical kernel removes small debris, noise pixels, and thin connective tissue threads that passed the luminance threshold.

7px kernel
03

Per-Component Crack Filling

Each connected component receives independent hole-filling via scipy binary_fill_holes — internal myofibril striations and fixation cracks disappear without merging adjacent fibers.

KEY STEP
04

Connected Component Analysis

Labeled components are filtered by area bounds derived from expected fiber size at the given scale. Components below the minimum are discarded as debris.

Area filter
05

Adaptive Watershed Splitting

Blobs exceeding 2× the median single-fiber area are split using distance-transform watershed. Sigma and min_distance adapt to each blob's estimated radius — not fixed constants.

Adaptive params
06

Shape Quality Filter

Each candidate is rejected if circularity < 0.18, aspect ratio > 5.5, or solidity < 0.62. Thresholds are tuned for large irregular manatee fibers.

QC filter
07

Morphometry & Statistics

Accepted fibers are measured: area (µm²), equivalent diameter (µm), circularity, aspect ratio, and solidity. Full statistical figures and CSV output are generated automatically.

Whole-Slide Intelligence

16,000+ fibers. One click.

Koniscope's SVS engine automatically parses entire gigapixel whole-slide images from end to end — no tile selection, no manual ROI drawing, no human input required after pressing Run.

16K+
Fibers Per SVS Scan

A single whole-slide SVS file can contain tens of thousands of individual muscle fibers. Koniscope detects and precisely outlines every one of them automatically.

Auto
Full Slide Parsing

Koniscope automatically selects the correct pyramid level closest to 20× magnification, tiles the entire slide, and dispatches each tile to a parallel worker — fully hands-free.

±µm
Micron-Level Precision

Every detected fiber is measured to sub-micron precision — diameter, area, circularity, aspect ratio, and solidity — using the scanner's native MPP (microns per pixel) calibration.

How SVS Auto-Parsing Works
①
LOAD SVS
OpenSlide reads the file and extracts pyramid metadata and MPP calibration
②
AUTO-LEVEL
Automatically selects pyramid level closest to 20× — no manual zoom selection
③
TILE GRID
Entire slide is divided into a grid of overlapping tiles covering every region
④
PARALLEL
All tiles dispatched simultaneously across CPU cores via ProcessPoolExecutor
⑤
COMPILE
Results merged into unified statistics, CSV export, and full morphometry dashboard

Detection Profiles

7 stain profiles

Each profile applies the correct color-space strategy for that staining protocol. Select from the dropdown — no manual threshold tuning needed.

Profile Detection Mode Best For
H&E (Standard) LAB L-channel Most H&E slides — recommended default
H&E (Pale / Weak) LAB L-channel Lightly stained or faded slides
Haematoxylin Only b-channel inv. Blue haematoxylin without eosin counterstain
NADH-TR Blackhat morph. Dark fibers on pale background
ATPase Blackhat morph. Type I / Type II fiber contrast staining
